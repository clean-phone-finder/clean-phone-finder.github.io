QUERY PARAMS І СТАН

Стан усіх вибраних значень зберігається в query params URL. Параметр phone може містити одне або кілька значень через кому — наприклад phone=iPhone+16,Pixel+9. Параметри country і lang містять одне значення або відсутні. Всі параметри необов'язкові. При перезавантаженні сторінки стан відновлюється з query params і відображається відповідний екран. Якщо query params відсутні — показується екран 1.

ДИЗАЙН-СИСТЕМА

Усі кольори, відступи і типографіка визначаються як CSS custom properties у блоці :root. Заборонено використовувати хардкодні значення кольорів будь-де в CSS, крім блоку :root і [data-theme="dark"]. Перемикання теми здійснюється через атрибут data-theme="dark" на елементі <html>. Стан теми зберігається в localStorage. Перемикач теми розташований у верхньому правому куті на кожному екрані.

:root {
  --bg: #ffffff;
  --surface: #f5f5f5;
  --border: #e0e0e0;
  --text-primary: #111111;
  --text-secondary: #666666;
  --accent: #2563eb;
  --accent-hover: #1d4ed8;
  --accent-text: #ffffff;
  --radius: 8px;
  --font: system-ui, -apple-system, BlinkMacSystemFont, sans-serif;
}

[data-theme="dark"] {
  --bg: #0f0f0f;
  --surface: #1a1a1a;
  --border: #2a2a2a;
  --text-primary: #f0f0f0;
  --text-secondary: #999999;
  --accent: #3b82f6;
  --accent-hover: #2563eb;
  --accent-text: #ffffff;
}

Типографіка: font-family — var(--font), base font size — 16px, line-height — 1.6. h1 — 1.75rem, font-weight 600. Підзаголовок — 1rem, font-weight 400, колір var(--text-secondary). Кнопки — 0.875rem, font-weight 500. Textarea — 0.9rem, line-height 1.7, font-family monospace.

Кнопки-чіпи використовують var(--surface) як фон, var(--border) як рамку, var(--text-primary) як колір тексту. Активний чіп використовує var(--accent) як фон і var(--accent-text) як колір тексту. Усі інтерактивні елементи використовують var(--radius) для border-radius. Transitions на color і background-color — 0.2s ease. Жодних box-shadow, крім тонкої рамки 1px на поверхнях через var(--border).

Прогрес-бар зверху кожного екрана висотою 2px, колір var(--accent), заповнення відповідно до поточного кроку: екран 1 — 25%, екран 2 — 50%, екран 3 — 75%, екран 4 — 100%.

ЕКРАН 1 — ВИБІР СМАРТФОНА

Вгорі h1: "Find a Smartphone Without Ads, Bloatware & Spyware". Під ним підзаголовок: "Generate a research prompt for AI based on real Reddit reviews and user reports". Індикатор кроку: "STEP 1 OF 4 · SMARTPHONE". Підказка під індикатором: "Select one to analyze or multiple to compare".

Кнопки-чіпи генеруються з CONFIG.phones.chips. Підтримується мульти-селект — можна обрати будь-яку кількість чіпів одночасно. Повторне натискання на активний чіп знімає виділення. Остання кнопка — "Other". При натисканні "Other" під кнопками з'являється текстове поле з HTML datalist, підказки для якого генеруються з CONFIG.phones.autocomplete. Значення введене в поле Other додається до списку обраних моделей і не замінює вибір чіпів — тобто можна одночасно обрати чіпи іввести модель через Other. Кнопка "Skip" пропускає крок без вибору. Кнопка "Next" переходить на екран 2 і зберігає всі обрані значення у query param phone через кому.

ЕКРАН 2 — ВИБІР КРАЇНИ

Індикатор кроку: "STEP 2 OF 4 · COUNTRY". Кнопки-чіпи генеруються з CONFIG.countries.chips. Одиночний селект — можна обрати лише одну країну. Остання кнопка — "Other" з текстовим полем і HTML datalist з підказками з CONFIG.countries.autocomplete. Кнопка "Skip". Кнопка "Back" повертає на екран 1 зі збереженням стану. Кнопка "Next" переходить на екран 3 і зберігає вибір у query param country.

ЕКРАН 3 — ВИБІР МОВИ ВІДПОВІДІ

Індикатор кроку: "STEP 3 OF 4 · LANGUAGE". Кнопки-чіпи генеруються з CONFIG.languages.chips — відображається поле label. Одиночний селект. При завантаженні екрана визначається мова браузера через navigator.language і зіставляється з полем code у CONFIG.languages.chips. Якщо збігається — ця кнопка отримує мітку "(your browser language)" і підсвічується як рекомендована. Якщо не збігається — жодна кнопка не підсвічується. Остання кнопка — "Other" з текстовим полем і HTML datalist з підказками з CONFIG.languages.autocomplete — відображаються значення поля label. Кнопка "Skip". Кнопка "Back" повертає на екран 2. Кнопка "Next" переходить на екран 4 і зберігає у query param lang значення поля label обраної мови.

ЕКРАН 4 — РЕЗУЛЬТАТ

Індикатор кроку: "STEP 4 OF 4 · YOUR PROMPT". Summary обраних параметрів: Smartphone (або Smartphones якщо обрано більше одного), Country, Language. Якщо параметр не обраний — відображається "Not specified". Якщо обрано кілька смартфонів — відображаються всі через кому. Далі textarea у режимі read-only зі згенерованим промптом. Textarea автоматично підлаштовує висоту під вміст. Кнопка "Copy Prompt" копіює вміст textarea в буфер обміну і тимчасово змінює текст на "Copied!" на 2 секунди. Кнопка "Start Over" скидає всі query params і повертає на екран 1. Кнопка "Back" повертає на екран 3. Нижче textarea — секція AI-асистентів, згенерована з CONFIG.ai_assistants. Нижче — секція посилань, згенерована з CONFIG.placeholder_links. Внизу — посилання на GitHub з CONFIG.github_url.

Посилання з img-іконками і назвами генеруються з CONFIG.ai_assistants. Іконки відображаються розміром 32x32px. SVG-іконки мають коректно відображатись в обох темах. Посилання відкриваються у новій вкладці.

ЛОГІКА ГЕНЕРАЦІЇ ПРОМПТУ

Згенерований промпт має бути суцільним текстом без emoji, без HTML, без Markdown і без будь-якого спеціального форматування. Промпт розбивається на логічні абзаци, розділені порожнім рядком, для зручного читання людиною. У кожному варіанті промпту має бути жорстка заборона для AI вигадувати факти, робити припущення без джерел або подавати неперевірену інформацію. Промпт має вимагати опори лише на реальні користувацькі звіти, обговорення і публічні джерела. Reddit є обов'язковим, але не єдиним джерелом. Якщо мова вказана — промпт містить явну вимогу надати відповідь цією мовою. Якщо мова не вказана — відповідь англійською.

Промпт завжди має включати перевірку таких аспектів:

Hardware-проблеми: перегрівання під навантаженням і у звичайному використанні, throttling процесора і GPU після тривалого навантаження, деградація акумулятора, нестабільність компонентів, проблеми з якістю збірки.

Software-проблеми: зависання і краші застосунків, нестабільні оновлення прошивки, баги після оновлень, деградація продуктивності з часом, проблеми з оперативною пам'яттю і управлінням фоновими процесами.

Реклама і приватність: наявність вбудованої реклами в системному інтерфейсі, spyware і adware у стоковій прошивці, примусове встановлення застосунків без згоди користувача, неможливість відключити або видалити нав'язливі системні застосунки, збір і передача даних користувача без явної згоди.

Сенсори і апаратні компроміси: наявність віртуального (програмного) датчика наближення замість апаратного інфрачервоного сенсора. Промпт має явно вимагати перевірки, чи використовує пристрій програмну емуляцію датчика наближення через акселерометр або датчик освітленості замість виділеного апаратного IR-сенсора, і якщо так — описати реальні скарги користувачів на некоректну поведінку екрана під час дзвінків.

Регіональні відмінності: відмінності прошивок для різних ринків, регіональні обмеження функцій, наявність додаткового bloatware в регіональних версіях.

Час присутності на ринку: якщо модель менше одного року на ринку — промпт має явно вимагати попередити користувача про недостатню кількість незалежних довгострокових відгуків.

Альтернативи: промпт має вимагати запропонувати конкретні альтернативні моделі без описаних проблем.

Варіанти промпту залежно від заповнених полів:

Якщо не вказано нічого (або лише мова): загальний промпт для worldwide вибору смартфона на основі реальних обговорень.

Якщо вказано лише країну (або країна + мова): промпт враховує регіональні особливості, доступність моделей, регіональні прошивки, обмеження та специфічні скарги користувачів з цього регіону.

Якщо вказано одну модель (або одна модель + країна, або одна модель + мова, або всі три поля з однією моделлю): промпт фокусується на аналізі цієї конкретної моделі з перевіркою всіх аспектів зі списку вище, з урахуванням країни якщо вказана.

Якщо вказано кілька моделей: промпт вимагає порівняльного аналізу всіх вказаних моделей. Для кожного аспекту зі списку вище AI має порівняти моделі між собою на основі реальних користувацьких звітів і публічних обговорень, а не абстрактних характеристик. Порівняння має бути структуроване по аспектах, а не по моделях окремо — тобто спочатку розглядається hardware по всіх моделях, потім software по всіх моделях, і так далі. В кінці має бути підсумкова рекомендація яка модель є кращим вибором і чому, з урахуванням країни якщо вказана. Якщо для деяких моделей недостатньо незалежних відгуків — AI має явно про це попередити.
