TECHNICAL REQUIREMENTS

The file must be a single self-contained HTML file with embedded CSS and JavaScript. No CSS frameworks, JavaScript frameworks, libraries, or CDNs are allowed. The page must be fully static and responsive. SVG icons for AI assistants must render correctly in both themes.

DATA — JSON CONFIGURATION

All data lists must be defined as JSON objects at the top of the JavaScript code under a comment CONFIG — EDIT HERE and must not be duplicated anywhere else in the code. When these JSON objects are changed, the page must automatically reflect the updated data without any changes to the rest of the code.

Each classifier (phones, countries, languages) has two lists. The first — chips — contains items displayed as chip buttons. The second — autocomplete — contains the full extended list of all known values for that classifier, including those already in chips. The Other field for each classifier uses the autocomplete list for suggestions via HTML datalist. Items from chips are not removed from autocomplete — they are also present there, since the user may want to type them manually.

QUERY PARAMS AND STATE

The state of all selected values is stored in URL query params. The phone param may contain one or multiple values separated by commas — for example phone=iPhone+16,Pixel+9. The country and lang params contain a single value or are absent. All parameters are optional. When the page is reloaded, state is restored from query params and the corresponding screen is shown. If no query params are present, screen 1 is shown.

DESIGN SYSTEM

All colors, spacing, and typography values must be defined as CSS custom properties on the :root element. Hardcoded color values are forbidden anywhere in CSS except inside the :root and [data-theme="dark"] blocks. Theme switching is handled via the data-theme="dark" attribute on the <html> element. Theme state is saved in localStorage. The theme toggle is positioned in the top right corner on every screen.

PROMPT GENERATION LOGIC

The generated prompt must be plain text with no emoji, no HTML, no Markdown, and no special formatting of any kind. The prompt is divided into logical paragraphs separated by blank lines for comfortable human reading. Every prompt variant must include a strict prohibition against the AI inventing facts, making unsourced assumptions, or presenting unverified information. The prompt must require relying only on real user reports, discussions, and public sources. Reddit is a mandatory but not exclusive source. If a language is specified — the prompt must explicitly require the AI to respond in that language. If no language is specified — the response must be in English.

The prompt must always include verification of the following aspects:

Hardware issues: overheating under load and during normal use, CPU and GPU throttling after sustained load, battery degradation, component instability, build quality issues.

Software issues: app freezes and crashes, unstable firmware updates, post-update bugs, performance degradation over time, RAM management and background process handling problems.

Ads and privacy: built-in advertising in the system interface, spyware and adware in stock firmware, forced app installation without user consent, inability to disable or remove intrusive system apps, collection and transmission of user data without explicit consent.

Sensors and hardware compromises: presence of a virtual (software-based) proximity sensor instead of a dedicated hardware infrared sensor. The prompt must explicitly require checking whether the device uses software emulation of the proximity sensor via accelerometer or ambient light sensor instead of a dedicated hardware IR sensor, and if so — describe real user complaints about incorrect screen behavior during phone calls.

Regional differences: firmware differences across markets, regional feature restrictions, presence of additional bloatware in regional versions.

Time on market: if the model has been on the market for less than one year — the prompt must explicitly require warning the user about insufficient independent long-term reviews.

Alternatives: the prompt must require suggesting specific alternative models without the described issues.

Prompt variants depending on filled fields:

If nothing is specified (or only language): a general worldwide prompt for choosing a smartphone based on real user discussions.

If only country is specified (or country + language): the prompt accounts for regional specifics, model availability, regional firmware, restrictions, and user complaints specific to that region.

If one smartphone is specified (with or without country and language): the prompt focuses on analyzing that specific model checking all aspects from the list above, taking the country into account if specified.

If multiple smartphones are specified: the prompt requires a comparative analysis of all specified models. For each aspect from the list above, the AI must compare the models against each other based on real user reports and public discussions — not abstract specifications. The comparison must be structured by aspect rather than by model — meaning hardware is covered across all models first, then software across all models, and so on. At the end there must be a summary recommendation of which model is the better choice and why, taking the country into account if specified. If independent long-term reviews are insufficient for any of the models — the AI must explicitly state this.
